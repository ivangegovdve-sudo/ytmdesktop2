## 2024-05-15 - [CRITICAL] Disable nodeIntegration in main views to prevent RCE
**Vulnerability:** `nodeIntegration: true` was enabled in `createView` and `createPopup` in `src/main/utils/view.ts` and not explicitly disabled for `createYoutubeView` in `src/main/utils/windowManager.ts`. Since `contextIsolation` was also set to `false`, this would allow any remote code (like an XSS on YouTube Music) full access to Node APIs, leading to RCE.
**Learning:** In Electron, views that load remote content MUST have `nodeIntegration: false`.
**Prevention:** Always ensure `nodeIntegration` is `false` (which is the default in newer Electron versions, but explicitly disabling it is best when migrating or overriding defaults) and use `contextIsolation: true` with a safe preload script.

## $(date +%Y-%m-%d) - [High] Fix path traversal vulnerability in app.openFile IPC handler
**Vulnerability:** The `app.openFile` IPC handler accepted arbitrary file paths from the renderer process. It attempted to secure this by checking if the path ended in allowed extensions (`.txt`, `.css`, etc.) using a Regex, but failed to restrict the base directory. An attacker who compromises the renderer process could potentially exploit this to read/open arbitrary local files matching these extensions on the user's filesystem (e.g. `../../../../../etc/passwd.txt` or `.bat` files if the Regex could be bypassed).
**Learning:** Checking file extensions is not sufficient protection for handlers that use dangerous methods like `shell.openPath()`. Using `string.startsWith(allowedPath)` without a trailing slash to validate directories introduces a directory prefix bypass vulnerability (e.g., `C:\Documents\ytmdesktop_malicious`). Furthermore, directly invoking `this.app` on classes implementing `BaseProvider` results in crashes; always use the explicitly injected `app` instance or the `import { app } from "electron"` module directly.
**Prevention:** Always restrict paths to explicitly defined safe directories using `path.relative(allowedPath, targetPath)`. A target path is securely within the `allowedPath` only if the resulting relative path does not start with `..` and is not an absolute path (`!path.isAbsolute(relativePath)`).
