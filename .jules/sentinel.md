## 2024-05-18 - [CRITICAL] Misconfigured Electron Protocol bypasses CSP
**Vulnerability:** The application was registering standard protocols (`http` and `https`) using `protocol.registerSchemesAsPrivileged` with `bypassCSP: true`. This globally disables Content Security Policy (CSP) enforcement for resources loaded over these schemes, exposing the application to widespread Cross-Site Scripting (XSS) vulnerabilities.
**Learning:** `bypassCSP: true` is a high-risk property. Developers sometimes use it to circumvent strict default CSPs or CORS restrictions during rapid feature development, without realizing its profound global impact on the application's security posture. Disabling CSP allows malicious inline scripts, external script injection, and eval(), drastically weakening defense-in-depth mechanisms.
**Prevention:** Never use `bypassCSP: true` unless there is a strictly documented and architectural necessity that cannot be solved by adjusting the CSP headers themselves. If specific external resources are needed, properly configure `Content-Security-Policy` headers to allowlist them rather than removing the entire policy constraint.
