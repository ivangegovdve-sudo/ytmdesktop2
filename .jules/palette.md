## 2024-05-24 - Window Controls Accessibility
**Learning:** Core application components like title bar window controls (Minimize, Maximize, Close) are frequently implemented as unsemantic `<div>`s instead of native `<button>`s, depriving users of keyboard navigability, focus states, and essential `aria-label` context for screen readers. Since these controls represent the absolute most critical interactions for window management, ensuring their accessibility is vital.
**Action:** When auditing or implementing custom application chrome, always inspect custom window controls and title bars. Ensure they use semantically correct interactive elements (`<button>`) and always include an explicit `aria-label` attribute describing their function for assistive technologies, particularly when they consist exclusively of icons.
